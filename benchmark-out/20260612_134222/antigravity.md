# Python Inline Cookbooks Data Module

The module `extensions/copilot/src/extension/prompts/node/inline/pythonCookbookData.ts` is a static data provider within the VS Code Copilot extension. It holds structured documentation, explanations, and before/after code examples for Python linting rules and code style recommendations, primarily focusing on rules implemented by **Ruff** (a fast Python linter and formatter).

---

## Purpose and Functionality

The main purpose of this module is to supply the VS Code Copilot extension with quick-fix cookbooks and descriptions for specific Python diagnostics or rule violations. When a user runs into a specific diagnostic error code (such as a Ruff rule violation), Copilot can reference this registry to:
1. Explain the violation in human-readable terms.
2. Display a concrete code comparison showing the incorrect pattern (`[Before]`) and the corrected pattern (`[After]`).
3. Offer automated or guided inline quick-fixes based on these cookbook definitions.

> [!NOTE]  
> This file contains the comment `// this file is automatically generated. Do not edit it.`. It is generated programmatically during the build or code-generation process from a central registry of rule definitions to keep it synchronized with the linter specifications.

---

## Module Structure and Exports

### 1. Types
* **`PromptMap`**: A simple type alias defined as `Record<string, string>`. It maps unique rule identifiers to their respective cookbook strings.
  ```typescript
  type PromptMap = Record<string, string>;
  ```

### 2. Exports
* **`pythonRuffCookbooks`**: An object of type `PromptMap` containing detailed instructions and examples for various Python rules.

---

## Key Categories of Represented Rules

The map covers multiple code quality, type-safety, and framework-specific guidelines:

| Rule Code Prefix | Domain / Tool | Examples Covered |
| :--- | :--- | :--- |
| **`AIR`** | Apache Airflow | `AIR001` (variable naming matching `task_id`), `AIR301` (explicit DAG scheduling), `AIR302` (deprecated `days_ago`), `AIR303` (`FabAuthManager` import migration). |
| **`ERA`** | Eradicate | `ERA001` (detecting and removing dead/commented-out code). |
| **`FAST`** | FastAPI | `FAST001` (redundant `response_model`), `FAST002` (migrating from `Depends` to `Annotated`), `FAST003` (missing path parameters). |
| **`YTT`** | flake8-2020 | `YTT101` - `YTT303` (incorrect system version checks using `sys.version` string slicing; recommending tuple comparison via `sys.version_info`). |
| **`ANN`** | flake8-annotations | `ANN001` - `ANN102` (missing type annotations for arguments, `*args`, `**kwargs`, `self`, and `cls`). |

---

## Data Format Pattern

Each cookbook entry in the `pythonRuffCookbooks` record adheres to a strict text-based template:

```
[Rule Description]

[Before]
```python
[Non-compliant Code Snippet]
```

[After]
```python
[Compliant Code Snippet]
```
```

This structured text allows the Copilot extension to easily parse or inject the descriptions and examples directly into UI panels, hover widgets, or prompt contexts for LLM generation.

---

## How It Fits Into the Project

Within the broader architecture of the VS Code Copilot extension:
1. **Diagnostic Integration**: When a Python file is open, VS Code displays diagnostic warnings/errors (e.g., from Python extension, Ruff, or flake8).
2. **Context Resolution**: The Copilot extension listens to diagnostic events. If the error ID matches one of the keys in `pythonRuffCookbooks` (e.g., `YTT101`), the extension retrieves the corresponding cookbook entry.
3. **Prompt Enrichment**:
   - The retrieved string is injected into the LLM system prompt as few-shot examples or direct context.
   - Alternatively, it is presented as local hover documentation or quick-fix actions directly within the editor interface, reducing LLM token consumption by using pre-compiled local rule mappings.
